# iOS HealthKit setup (Xcode + native build)

The app now integrates directly with Apple HealthKit through the Capacitor
plugin `@perfood/capacitor-healthkit` (v1.3.2). V1 scope is pure observations:
we read 6 data types (sleep, resting HR, HRV, heart rate, active energy,
workouts) and forward them to the `wearable-ingest` edge function, which
updates the existing `wearable_daily_summary` via `recompute_wearable_summary`.
No score, no recommendation, no color coding is derived from HealthKit data.

## 1. Enable HealthKit capability in Xcode
1. Open the iOS project: `npx cap open ios`
2. Select the **App** target → **Signing & Capabilities**.
3. Click **+ Capability** → add **HealthKit**.
4. Leave **Clinical Health Records** and **Background Delivery** unchecked
   (not used in V1).

## 2. Info.plist usage strings
`ios/App/App/Info.plist` needs the **read** usage description. We do NOT
write to HealthKit, so `NSHealthUpdateUsageDescription` is not required.

```xml
<key>NSHealthShareUsageDescription</key>
<string>Sportstalent læser dine sundhedsdata (søvn, puls, HRV og træning) fra Apple Health for at vise din restitution og dokumentere din træning. Sportstalent reads your health data (sleep, heart rate, HRV and workouts) from Apple Health to show your recovery and log your training.</string>
```

## 3. Sync and run
```bash
git pull
npm install       # picks up @perfood/capacitor-healthkit
npm run build
npx cap sync ios
npx cap open ios
```
Then press ▶ in Xcode with your iPhone connected. The Podfile is
auto-generated by `npx cap sync ios`; no manual Podfile edits are needed.

## 4. In the app
- Settings → **Health** → **Connect Apple Health** (button only shown on
  iOS native).
- The first sync pulls 90 days of history; subsequent syncs pull the last
  30 days. Sync is throttled to at most once per hour and runs
  automatically on app open and on app resume.

## 5. Shortcut path still works
The legacy iOS Shortcut → `health-sync-simple` → `resync-health` path is
untouched. Users who don't grant HealthKit access keep the Shortcut flow
as before. For users with both paths active, HealthKit wins: since it
populates `wearable_samples`, `recompute_wearable_summary` re-aggregates
`wearable_daily_summary` from those samples.

## 6. Dev hot-reload vs native build
Do NOT re-add `server.url` to `capacitor.config.ts` when testing HealthKit —
HealthKit only bridges reliably when the web layer is loaded from inside
the app bundle.
