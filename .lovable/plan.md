# Plan: Capacitor iOS setup for Apple Watch / HealthKit (Steps 1–4)

This plan covers ONLY the codebase changes. After approval, you'll run the Mac/Xcode steps separately (export to GitHub → `npm install` → `npx cap add ios` → `npx cap sync` → open in Xcode).

## What I'll change in the project

### Step 1 — Install Capacitor dependencies
Add to `package.json`:
- `@capacitor/core`
- `@capacitor/ios`
- `@capacitor/cli` (devDependency)
- `capacitor-health` (community HealthKit + Health Connect plugin — already matches the registry names `CapacitorHealth` / `HealthConnect` our `src/lib/wearables/index.ts` looks for)

### Step 2 — Create `capacitor.config.ts` at project root
```ts
import type { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'app.lovable.a65f5c861a844640b1394767189347ea',
  appName: 'Sportstalent',
  webDir: 'dist',
  server: {
    url: 'https://a65f5c86-1a84-4640-b139-4767189347ea.lovableproject.com?forceHideBadge=true',
    cleartext: true,
  },
  ios: {
    contentInset: 'always',
  },
};

export default config;
```
Note: the `server.url` enables hot-reload from the Lovable sandbox during development. Before App Store submission you'll remove that block so the app loads the bundled `dist/`.

### Step 3 — iOS HealthKit Info.plist usage strings
Add a small helper file `ios-healthkit-info.md` at project root with the exact `Info.plist` keys you must paste in Xcode (Lovable can't write into the iOS native folder because it's generated by `npx cap add ios` on your Mac):
- `NSHealthShareUsageDescription` — "Sportstalent reads your sleep, resting heart rate, HRV, steps and workouts from Apple Health to personalize training and recovery."
- `NSHealthUpdateUsageDescription` — "Sportstalent does not write data to Apple Health." (still required by Apple if HealthKit capability is enabled)

Also documents enabling the **HealthKit capability** in Xcode → Signing & Capabilities.

### Step 4 — Wire native sync into app lifecycle
Small edits to existing files (no architecture change — `src/lib/wearables/index.ts` already branches on platform and reads from `Capacitor.Plugins`):

- **`src/App.tsx`** — on app focus / resume, call the existing `syncOnAppOpen()` from `@/lib/wearables`. Already rate-limited to 30 min, already no-ops on web.
- **`src/pages/WearablesSettings.tsx`** — keep the existing web-fallback warning; on native iOS the existing "Connect Apple Health" button will now actually trigger HealthKit auth via the bundled plugin.
- **`src/main.tsx`** — no change needed; service worker block already excludes native (Capacitor uses `capacitor://` scheme).

No edits to `src/lib/wearables/index.ts` are required — it was built for exactly this plugin shape.

## What you'll do on your Mac afterward
1. Export to GitHub (Connectors → GitHub) and `git clone` locally.
2. `npm install`
3. `npx cap add ios`
4. `npx cap sync ios`
5. `npx cap open ios` → in Xcode: add HealthKit capability + paste the two Info.plist strings from `ios-healthkit-info.md`.
6. Plug in your iPhone, select it as run target, press ▶. Trust the developer cert on the phone (Settings → General → VPN & Device Management).
7. Open the app → Settings → Wearables → Connect Apple Health → grant permissions → workouts/sleep/HR/HRV/steps will start syncing.
